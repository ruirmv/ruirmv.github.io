Making a commit

How to write a good commit message?

  1. Use the imperative mood in the subject line. The subject line should complete the sentence: if applied, this commit will subject_line
  2. Separate the subject line from the body with a blank line
  3. Limit the subject line to arround 50 characters
  4. Capitalize the subject line
  5. Do not end the subject line with a period
  6. Wrap the body at around 72 characters
  7. Use the body to explain what and why a change was made instead of how

Example

  1. Implement auto login for new signups

  3. - Added password verification

  5. Resolves: #231

Making a pull request

Before you make a pull request...

  • Format the code: make format
  • Lint the code: make lint
  • Run the test unit: make test-unit
  • Check the definition of delivery (use checklist)

Keep feature branch up to date with master

  1. git checkout master
  2. git pull
  3. git checkout -
  4. git rebase master
