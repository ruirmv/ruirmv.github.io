Workflow

  1. Create a virtual environment and install Django.
  2. Establish a Django project - django-admin startproject mysite. More on django-admin and manage.py on django-admin and manage.py. The first part of the tutorial contains an overview of the files generated by this command.
  3. Run the server - python manage.py runserver [ip:][port]. The server should not be used in production.
  4. Creating an app - python manage.py startapp app_name. A project might contain multiple apps. Additionally, create a urls.py inside the app, to map between URLs and views.
  5. Register the app in settings.INSTALLED_APPS.
  6. Link the app URLs with the project URLs.

Admin

Creating an admin user

python manage.py createsuperuser

Enter the admin site

Simply run the server and go to "/admin/.

The contents displayed for Group and Users are provided by django.contrib.auth, the module responsible for user authentication.

Managing URLs

The file urls.py provides a connection between URLs and views.

A nice overview of managing URLs is in the first part of the tutorial.

path(route, view, name) # name lets you refer to it unambiguously from elsewhere in Django

Creating views

The simplest way to create a view is with a basic HttpResponse.

from django.http import HttpResponse def index(request: HttpRequest): return HttpResponse("Hello")
Note: views are responsible for sending an HttpResponse or raise an Http exception.

Templates

The first step to create a template in an app is to create a folder named templates inside the app and inside of it another folder with the same name as the app app_name/templates/app_name.

Models

Models are basically the database layout with additional metadata.

It contains the fields and behaviors of the data it stores.

See the documentation for a complete information about models.

Each model has a number of class attributes, each representing a database field. Relationships between tables are also defined here.

from django.db import models class Model(models.Model) field = models.Field(...)

For more information about fields, such as field types, options and relationships check the reference.

Creating or changing a model

  1. Change/create the models.
  2. Run python manage.py makemigrations to create migrations for those changes.
  3. Run python manage.py migrate to apply those changes to the database.
